## 什么是部署？为什么要部署？

### 什么是部署？

**部署**就是把你的网站从你的电脑搬到互联网上，让全世界的人都能访问。

- **现在**：只有你的电脑能访问 `http://localhost:3000`
- **部署后**：任何人都可以通过网址（如 `https://www.yourwebsite.com`）访问你的网站

### 为什么要部署？

1. ✅ **让客户能访问**：客户不需要来你家，就能看到你的网站
2. ✅ **24小时在线**：网站一直运行，不会因为你关电脑而停止
3. ✅ **专业形象**：有自己的域名和网站，更专业
4. ✅ **SEO优化**：搜索引擎能找到你的网站

## 部署前准备

### ✅ 检查清单

在开始部署前，请确认：

- [ ] 你的网站在本地能正常运行（`http://localhost:3000` 能打开）

## 方法一：使用云服务器部署（推荐）

### 第一步：购买云服务器

#### 1.1 选择云服务商

推荐平台：
- **阿里云**：https://www.aliyun.com（国内访问快）
- **腾讯云**：https://cloud.tencent.com（性价比高）
- **华为云**：https://www.huaweicloud.com

#### 1.2 购买服务器

**以阿里云为例：**

1. 注册/登录阿里云账号
2. 进入 **云服务器ECS** 页面
3. 点击 **立即购买**
4. 选择配置：
   - **地域**：选择离你最近的（如：华东1-杭州）
   - **实例规格**：选择 **1核2GB**（够用，约¥50/月）
   - **镜像**：选择 **Ubuntu 22.04** 或 **CentOS 7**
   - **系统盘**：40GB（默认即可）
   - **网络**：选择 **按量付费** 或 **包年包月**
5. 设置密码（**重要！记住这个密码**）
6. 点击 **立即购买** 并支付

#### 1.3 获取服务器信息

购买成功后，你会得到：
- **公网IP**：例如 `47.xxx.xxx.xxx`（这是你服务器的地址）
- **用户名**：通常是 `root`
- **密码**：你刚才设置的密码

---

### 第二步：连接服务器

#### 2.1 Windows 用户

1. **下载 PuTTY**（免费软件）
   - 下载地址：https://www.putty.org/
   - 下载 `putty.exe` 文件

2. **打开 PuTTY**
   - Host Name：输入你的 **公网IP**
   - Port：`22`
   - Connection type：选择 **SSH**
   - 点击 **Open**

3. **登录**
   - 输入用户名：`root`
   - 输入密码（输入时不会显示，直接输入后按回车）

#### 2.2 Mac 用户

1. **打开终端**（Terminal）
2. **输入命令**：
   ```bash
   ssh root@你的公网IP
   ```
   例如：`ssh root@47.xxx.xxx.xxx`
3. **输入密码**（输入时不会显示）

#### 2.3 连接成功标志

看到类似这样的提示，说明连接成功：
```
Welcome to Ubuntu 22.04 LTS
root@iZxxx:~#
```

### 第三步：在服务器上安装环境

#### 3.1 更新系统

```bash
# Ubuntu/Debian 系统
apt update && apt upgrade -y

# CentOS 系统
yum update -y
```

#### 3.2 安装 Node.js

```bash
# 方法一：使用 NodeSource（推荐）
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt-get install -y nodejs

# 或者使用 nvm（更灵活）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
```

#### 3.3 验证安装

```bash
node -v    # 应该显示 v18.x.x 或更高
npm -v     # 应该显示 9.x.x 或更高
```

### 第四步：上传项目文件

#### 4.1 方法一：使用 Git（推荐）

**在你的电脑上：**

1. **安装 Git**（如果还没安装）
   - Windows：https://git-scm.com/download/win
   - Mac：`brew install git`

2. **初始化 Git 仓库**（在你的项目文件夹里）
   ```bash
   cd "C:\Users\你的用户名\Desktop\JW 2"
   git init
   git add .
   git commit -m "Initial commit"
   ```

3. **创建 GitHub 仓库**
   - 访问 https://github.com
   - 点击右上角 **+** → **New repository**
   - 输入仓库名（如：`my-website`）
   - 点击 **Create repository**

4. **上传到 GitHub**
   ```bash
   git remote add origin https://github.com/你的用户名/my-website.git
   git push -u origin main
   ```

**在服务器上：**

```bash
# 安装 Git
apt install git -y

# 克隆项目
cd /root
git clone https://github.com/你的用户名/my-website.git
cd my-website
```

#### 4.2 方法二：使用 FTP 工具（简单但慢）

1. **下载 FileZilla**（免费）
   - 下载地址：https://filezilla-project.org/

2. **连接服务器**
   - 主机：`sftp://你的公网IP`
   - 用户名：`root`
   - 密码：你的服务器密码
   - 端口：`22`

3. **上传文件**
   - 左侧：你的项目文件夹
   - 右侧：服务器的 `/root` 目录
   - 拖拽整个项目文件夹到右侧

### 第五步：配置和启动项目

#### 5.1 进入项目目录

```bash
cd /root/my-website  # 或者你上传的文件夹名
```

#### 5.2 安装依赖

```bash
npm install
```

#### 5.3 创建环境变量文件

```bash
nano .env
```

**输入以下内容**（按 `Ctrl+X`，然后按 `Y`，再按 `Enter` 保存）：

```env
PORT=3000
NODE_ENV=production
JWT_SECRET=请改成至少32个字符的随机字符串，例如：MySuperSecretKey2024!@#$%^&*
JWT_EXPIRES_IN=24h
DB_DIALECT=sqlite
```

**生成随机密钥的方法**：
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

#### 5.4 初始化数据库

```bash
DB_DIALECT=sqlite node init-db.js
```

#### 5.5 测试启动

```bash
node app.js
```

如果看到：
```
服务器正在运行，端口: 3000
数据库连接成功！
```

说明启动成功！按 `Ctrl+C` 停止。

---

### 第六步：使用 PM2 保持运行

#### 6.1 安装 PM2

```bash
npm install -g pm2
```

#### 6.2 启动应用

```bash
cd /root/my-website
DB_DIALECT=sqlite pm2 start app.js --name my-website
```

#### 6.3 设置开机自启

```bash
pm2 startup
pm2 save
```

#### 6.4 常用命令

```bash
pm2 status              # 查看状态
pm2 logs my-website     # 查看日志
pm2 restart my-website  # 重启
pm2 stop my-website     # 停止
```

---

### 第七步：配置防火墙和域名

#### 7.1 开放端口

```bash
# Ubuntu/Debian
ufw allow 3000
ufw enable

# CentOS
firewall-cmd --permanent --add-port=3000/tcp
firewall-cmd --reload
```

#### 7.2 配置域名（可选）

1. **购买域名**
   - 阿里云：https://wanwang.aliyun.com
   - 腾讯云：https://dnspod.cloud.tencent.com

2. **解析域名**
   - 登录域名管理后台
   - 添加 **A记录**：
     - 主机记录：`@` 或 `www`
     - 记录值：你的 **公网IP**
     - TTL：`600`

3. **使用 Nginx 反向代理**（推荐）

```bash
# 安装 Nginx
apt install nginx -y

# 创建配置文件
nano /etc/nginx/sites-available/my-website
```

**输入以下内容**（替换 `yourdomain.com` 为你的域名）：

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**启用配置**：

```bash
ln -s /etc/nginx/sites-available/my-website /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

## 方法二：使用免费托管平台（最简单）

### 🚀 方案一：Railway（推荐，最简单）

#### 步骤 1：注册账号

1. 访问 https://railway.app
2. 点击 **Start a New Project**
3. 使用 **GitHub** 账号登录（如果没有，先注册 GitHub）

#### 步骤 2：准备项目

**在你的电脑上：**

1. **创建 `.env.example` 文件**（在项目根目录）：
   ```env
   PORT=3000
   NODE_ENV=production
   JWT_SECRET=your-secret-key
   JWT_EXPIRES_IN=24h
   DB_DIALECT=sqlite
   ```

2. **创建 `Procfile` 文件**（在项目根目录）：
   ```
   web: node app.js
   ```

3. **上传到 GitHub**（参考方法一的第四步）

#### 步骤 3：部署到 Railway

1. **在 Railway 中创建项目**
   - 点击 **New Project**
   - 选择 **Deploy from GitHub repo**
   - 选择你的仓库

2. **配置环境变量**
   - 点击项目 → **Variables**
   - 添加以下变量：
     ```
     PORT=3000
     NODE_ENV=production
     JWT_SECRET=你的随机密钥（至少32字符）
     JWT_EXPIRES_IN=24h
     DB_DIALECT=sqlite
     ```

3. **部署**
   - Railway 会自动检测并部署
   - 等待 2-5 分钟

4. **获取访问地址**
   - 部署成功后，点击 **Settings** → **Generate Domain**
   - 你会得到一个地址，如：`https://my-website.up.railway.app`

#### 步骤 4：初始化数据库

Railway 会自动运行，但数据库需要初始化。你需要：

1. **添加启动脚本**（在 `package.json` 中）：
   ```json
   {
     "scripts": {
       "start": "node app.js",
       "postinstall": "DB_DIALECT=sqlite node init-db.js || true"
     }
   }
   ```

2. **重新部署**（Railway 会自动检测并重新部署）

## 常见问题解决

### ❌ 问题 1：网站打不开

**可能原因：**
- 服务器没有启动
- 防火墙没有开放端口
- 域名没有解析

**解决方法：**

```bash
# 检查服务是否运行
pm2 status

# 如果没有运行，启动它
pm2 start app.js --name my-website

# 检查端口是否开放
netstat -tlnp | grep 3000

# 检查防火墙
ufw status
```

---

### ❌ 问题 2：数据库错误

**错误信息：** `数据库连接失败`

**解决方法：**

```bash
# 检查数据库文件是否存在
ls -la config/database.sqlite

# 如果不存在，重新初始化
DB_DIALECT=sqlite node init-db.js

# 检查文件权限
chmod 644 config/database.sqlite
```

---

### ❌ 问题 3：端口被占用

**错误信息：** `EADDRINUSE: address already in use`

**解决方法：**

```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程（替换 PID 为实际进程号）
kill -9 PID

# 或者修改端口（在 .env 文件中）
PORT=3001
```

---

### ❌ 问题 4：内存不足

**错误信息：** `JavaScript heap out of memory`

**解决方法：**

```bash
# 增加 Node.js 内存限制
NODE_OPTIONS="--max-old-space-size=2048" pm2 restart my-website
```

---

### ❌ 问题 5：上传文件失败

**可能原因：**
- `uploads/` 目录不存在
- 目录权限不足

**解决方法：**

```bash
# 创建上传目录
mkdir -p uploads

# 设置权限
chmod 755 uploads
```
