# 图片上传功能使用指南

## 🚀 启动本地服务器

### 步骤1：打开终端
打开 Mac 的"终端"应用（Terminal）

### 步骤2：启动服务器
在终端中运行：

```bash
cd "/Users/zade/Desktop/JW 2"
DB_DIALECT=sqlite node app.js
```

### 步骤3：等待启动成功
看到以下信息表示启动成功：
```
上传目录: /Users/zade/Desktop/JW 2/uploads
服务器正在运行，端口: 3000
访问地址: http://localhost:3000
API地址: http://localhost:3000/api
数据库模型同步成功
```

## 📍 访问地址

启动成功后，在浏览器中打开：

**地址**: `http://localhost:3000`

### 后台管理系统
**地址**: `http://localhost:3000/html/admin/dashboard-new.html`

- 密码: （你设置的密码）

## 🖼️ 图片上传功能位置

### 1. 网站信息管理（Logo、Favicon、首页背景）

**访问路径**：
1. 登录后台管理系统
2. 点击左侧菜单 **"网站信息"**
3. 或直接访问：`http://localhost:3000/html/admin/website-info.html`

**可以上传的图片**：
- ✅ **Logo** - 网站Logo图片
- ✅ **Favicon** - 浏览器标签页图标
- ✅ **首页头图背景** - 首页大图的背景图片

**使用方法**：
1. 找到对应的图片字段（如"Logo"）
2. 点击 **"选择文件"** 按钮
3. 选择图片文件（支持 jpg、png、gif、svg，最大5MB）
4. 上传成功后，图片路径会自动填入输入框
5. 点击 **"保存更改"** 按钮保存

---

### 2. 服务项目管理（服务图片）

**访问路径**：
1. 登录后台管理系统
2. 点击左侧菜单 **"服务项目"**
3. 或直接访问：`http://localhost:3000/html/admin/services.html`

**可以上传的图片**：
- ✅ **服务图片** - 每个服务项目的展示图片

**使用方法**：
1. 点击 **"添加服务项目"** 或编辑现有服务
2. 在弹窗中找到 **"服务图片"** 字段
3. 点击 **"选择文件"** 按钮
4. 选择图片文件
5. 上传成功后，点击 **"保存"** 按钮

---

## 🔍 如何测试上传功能

### 测试步骤：

1. **启动服务器**（见上方步骤）

2. **打开后台管理**
   - 访问：`http://localhost:3000/html/admin/dashboard-new.html`
   - 登录后台

3. **测试Logo上传**
   - 进入"网站信息管理"
   - 找到"Logo"字段
   - 点击"选择文件"
   - 选择一张图片（建议使用小尺寸的Logo图片）
   - 等待上传完成
   - 查看输入框是否自动填入了路径（如：`/uploads/file-xxx.png`）
   - 查看预览图片是否显示
   - 点击"保存更改"

4. **验证上传结果**
   - 刷新前台页面：`http://localhost:3000`
   - 查看Logo是否已更新

## ⚠️ 注意事项

1. **文件大小限制**：最大 5MB
2. **支持格式**：jpg、png、gif、svg、pdf、doc、docx、xls、xlsx
3. **必须登录**：上传功能需要管理员登录
4. **保存数据**：上传后记得点击"保存"按钮，否则不会保存到数据库

---

## 🐛 如果上传失败

### 检查清单：

1. **服务器是否运行**
   - 检查终端是否显示"服务器正在运行"
   - 访问 `http://localhost:3000/health` 应该返回 `{"status":"ok"}`

2. **是否已登录**
   - 确保已登录后台管理系统
   - 检查浏览器控制台（F12）是否有"未登录"错误

3. **文件格式和大小**
   - 确保文件格式支持
   - 确保文件大小不超过5MB

4. **查看错误信息**
   - 打开浏览器控制台（F12 → Console）
   - 查看是否有红色错误信息
   - 查看服务器终端是否有错误日志
