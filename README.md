# 咨询公司官网

## 项目简介

咨询公司官网是一个基于 Node.js + Express + MySQL 开发的企业官网，包含前端展示页面和后端管理系统。该项目支持企业信息管理、服务项目管理、成功案例管理、博客管理、客户评价管理等功能。

## 技术栈

### 后端

- Node.js：JavaScript 运行时环境
- Express：Web 框架
- MySQL：关系型数据库
- Sequelize：ORM 框架
- JWT：身份认证
- Multer：文件上传

### 前端

- HTML/CSS/JavaScript：前端页面开发
- Bootstrap：UI 框架
- jQuery：DOM 操作库
- Font Awesome：图标库

## 快速开始

### 1. 环境准备

- Node.js：v16 或更高版本
- MySQL：v8.0 或更高版本

### 2. 安装依赖

```bash
# 克隆项目
git clone <repository-url>

# 进入项目目录
cd consulting-company

# 安装依赖
npm install
```

### 3. 配置环境变量

1. 复制 `.env` 文件：
   ```bash
   cp .env .env.local
   ```

2. 编辑 `.env.local` 文件，修改以下配置：
   ```
   # 数据库配置
   DB_HOST=localhost
   DB_PORT=3306
   DB_USER=root
   DB_PASSWORD=root
   DB_NAME=consulting_company

   # 服务器配置
   PORT=3000
   HOST=localhost

   # JWT配置
   JWT_SECRET=your_jwt_secret_key
   ```

### 4. 初始化数据库

1. 创建数据库：
   ```sql
   CREATE DATABASE consulting_company CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

2. 导入数据库初始化脚本：
   ```bash
   mysql -u root -p consulting_company < database.sql
   ```

### 5. 启动项目

```bash
# 开发模式启动
npm run dev

# 生产模式启动
npm start

# 使用 PM2 启动（推荐）
npm run start:pm2
```

### 6. 访问项目

- 前端首页：http://localhost:3000
- 后端管理系统：http://localhost:3000/html/admin/index.html

## 开发说明

### 1. 数据库模型

数据库模型定义在 `models/` 目录下，使用 Sequelize ORM 框架。每个模型对应一个数据库表，包含表结构和关联关系定义。

### 2. 路由设计

路由定义在 `routes/` 目录下，采用 RESTful API 设计风格。主要路由包括：

- `/api/website-info`：网站信息管理
- `/api/home-content`：首页内容管理
- `/api/navigation`：导航栏管理
- `/api/services`：服务项目管理
- `/api/projects`：成功案例管理
- `/api/blogs`：博客管理
- `/api/testimonials`：客户评价管理
- `/api/partners`：合作伙伴管理
- `/api/about`：关于我们管理
- `/api/team`：团队成员管理
- `/api/contacts`：联系我们管理
- `/api/users`：用户管理

### 3. 中间件

中间件定义在 `middleware/` 目录下，主要包括：

- 身份认证中间件
- 权限验证中间件
- 错误处理中间件
- 日志中间件

### 4. 前端页面开发

前端页面开发采用传统的 HTML/CSS/JavaScript 方式，使用 Bootstrap 框架和 jQuery 库。前端页面主要包括：

- 首页：展示公司基本信息、服务项目、成功案例等
- 服务项目：展示公司提供的服务项目列表和详情
- 成功案例：展示公司完成的成功案例列表和详情
- 博客：展示公司发布的博客文章列表和详情
- 关于我们：展示公司简介、文化、发展历程等
- 团队成员：展示公司团队成员信息
- 联系我们：提供公司联系方式和联系表单

## 常见问题

### 1. 数据库连接失败

检查以下配置：
- 数据库主机地址、端口、用户名、密码是否正确
- 数据库用户是否有足够的权限
- 数据库服务是否正常运行

### 2. 前端页面无法访问

检查以下配置：
- 前端页面文件是否存在
- 静态文件服务配置是否正确
- 防火墙是否允许 HTTP/HTTPS 端口访问

### 3. API 请求失败

检查以下配置：
- API 路由是否正确
- 后端服务是否正常运行
- CORS 配置是否正确
- 前端 API 请求地址是否正确

### 4. 上传文件失败

检查以下配置：
- 文件上传目录是否存在且有写入权限
- 上传文件大小是否超过限制
- 文件类型是否符合要求
