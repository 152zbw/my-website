## 更新前准备

### 1. 检查本地代码状态

在开始更新前，确保：

- ✅ 本地代码已经测试过，没有明显的语法错误
- ✅ 知道修改了哪些文件
- ✅ 服务器可以正常连接
- ✅ 知道服务器的 IP 地址和登录密码

### 2. 备份服务器数据库（重要！）

**⚠️ 重要提示：** 如果更新涉及数据库结构变化，必须先备份数据库！

在服务器上执行：

```bash
# 连接到服务器
ssh root@你的服务器IP

# 进入项目目录
cd /root/my-website

# 备份数据库
cp data/database.sqlite data/database.sqlite.backup.$(date +%Y%m%d_%H%M%S)

# 查看备份文件（确认备份成功）
ls -lh data/database.sqlite.backup.*
```

**预期结果：** 会看到类似 `database.sqlite.backup.20240115_143022` 的文件

## 本地操作步骤

### 步骤 1：打开终端（Mac）

1. 按 `Command + 空格键` 打开 Spotlight 搜索
2. 输入 `终端` 或 `Terminal`
3. 按回车键打开终端

### 步骤 2：进入项目目录

在终端中输入：

```bash
cd "/Users/zade/Desktop/JW 2"
```

**说明：**
- `cd` 是切换目录的命令
- 路径要用引号括起来，因为路径中有空格
- 按回车执行命令

**验证：** 终端提示符应该显示类似 `/Users/zade/Desktop/JW 2` 的路径

### 步骤 3：查看修改的文件

在终端中输入：

```bash
git status
```

**预期输出：**
- 会显示所有修改过的文件（红色 `M` 标记）
- 会显示新添加的文件（绿色 `??` 标记）

**示例输出：**
```
On branch main
Your branch is up to date with 'origin/main'.

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
        modified:   html/single-service.html
        modified:   html/index.html

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        html/admin/home-features.html
```

**说明：**
- `modified` 表示文件被修改了
- `Untracked files` 表示新文件还没有被 Git 跟踪

### 步骤 4：添加所有修改到暂存区

在终端中输入：

```bash
git add -A
```

**说明：**
- `git add` 是将文件添加到暂存区的命令
- `-A` 表示添加所有修改（包括新文件、修改的文件、删除的文件）
- 这个命令不会显示输出，如果没报错就是成功了

**验证：** 再次运行 `git status`，应该看到文件变成绿色（表示已添加到暂存区）

### 步骤 5：提交修改

在终端中输入：

```bash
git commit -m "修复服务详情页面显示问题和清理行号标记"
```

**说明：**
- `git commit` 是提交修改的命令
- `-m` 后面跟的是提交信息，描述这次修改的内容
- 提交信息应该清晰说明修改了什么

**预期输出：**
```
[main 5c65b8f] 修复服务详情页面显示问题和清理行号标记
 50 files changed, 8318 insertions(+), 1569 deletions(-)
```

**说明：**
- `5c65b8f` 是这次提交的唯一 ID
- `50 files changed` 表示修改了 50 个文件
- `8318 insertions` 表示新增了 8318 行代码
- `1569 deletions` 表示删除了 1569 行代码

### 步骤 6：推送到远程仓库

在终端中输入：

```bash
git push
```

**说明：**
- `git push` 是将本地提交推送到远程仓库（GitHub/Gitee）的命令
- 如果是第一次推送，可能需要输入 GitHub/Gitee 的用户名和密码
- 如果配置了 SSH 密钥，则不需要输入密码

**预期输出：**
```
To https://github.com/152zbw/my-website.git
   6bdc204..5c65b8f  main -> main
```

**说明：**
- `6bdc204..5c65b8f` 表示从旧版本更新到新版本
- `main -> main` 表示推送到 main 分支

**如果提示需要认证：**

1. **使用 SSH 密钥（推荐）：**
   ```bash
   # 检查是否有 SSH 密钥
   ls -la ~/.ssh
   
   # 如果没有，生成 SSH 密钥
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
   
   # 复制公钥到剪贴板
   cat ~/.ssh/id_rsa.pub | pbcopy
   
   # 然后到 GitHub/Gitee 网站添加 SSH 密钥
   ```

2. **使用 HTTPS（需要输入密码）：**
   - 输入 GitHub/Gitee 用户名
   - 输入密码（或 Personal Access Token）

**✅ 本地操作完成！** 现在代码已经推送到远程仓库了。

## 服务器操作步骤

### 步骤 1：连接到服务器

在终端中输入：

```bash
ssh root@你的服务器IP
```

**说明：**
- `ssh` 是远程连接命令
- `root` 是服务器用户名（通常是 root）
- `@` 后面是服务器的 IP 地址

**示例：**
```bash
ssh root@47.98.123.456
```

**预期输出：**
```
The authenticity of host '47.98.123.456 (47.98.123.456)' can't be established.
ECDSA key fingerprint is SHA256:xxxxx.
Are you sure you want to continue connecting (yes/no)?
```

**操作：** 输入 `yes` 并按回车

**然后会提示输入密码：**
```
root@47.98.123.456's password:
```

**操作：** 输入服务器密码（输入时不会显示，直接输入后按回车）

**✅ 连接成功标志：** 终端提示符变成类似 `[root@iZxxxx ~]#` 的格式

### 步骤 2：进入项目目录

在服务器终端中输入：

```bash
cd /root/my-website
```

**说明：**
- `cd` 是切换目录的命令
- `/root/my-website` 是项目在服务器上的路径
- 如果项目路径不同，请替换为实际路径

**验证：** 运行 `pwd` 命令，应该显示 `/root/my-website`

### 步骤 3：查看当前代码状态

在服务器终端中输入：

```bash
git status
```

**预期输出（正常情况）：**
```
On branch main
Your branch is up to date with 'origin/main'.
nothing to commit, working tree clean
```

**其他可能的情况：**

**情况 1：有未提交的修改**
```
On branch main
Your branch is up to date with 'origin/main'.

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
	modified:   package-lock.json

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	env
```

**处理方式：**
- **`package-lock.json` 的修改：** 这是正常的，通常是依赖更新导致的，可以提交：
  ```bash
  git add package-lock.json
  git commit -m "更新依赖"
  git push
  ```
- **`env` 文件：** 这是环境变量文件，**不应该提交**（已在 `.gitignore` 中忽略）
  - 如果显示为未跟踪文件，说明 `.gitignore` 可能没有正确配置
  - 可以忽略，不会影响推送

**情况 2：分支落后于远程**
```
On branch main
Your branch is behind 'origin/main' by 2 commits, and can be fast-forwarded.
```

**处理方式：** 直接运行 `git pull` 更新即可

**说明：**
- 如果显示 `Your branch is behind 'origin/main'`，说明需要更新
- 如果显示 `working tree clean`，说明当前没有未提交的修改
- 如果有修改但不想提交，可以使用 `git restore <file>` 丢弃修改

### 步骤 4：拉取最新代码

在服务器终端中输入：

```bash
git pull
```

**说明：**
- `git pull` 是从远程仓库拉取最新代码的命令
- 这个命令会下载你在本地推送的最新代码

**预期输出：**
```
remote: Enumerating objects: 50, done.
remote: Counting objects: 100% (50/50), done.
remote: Compressing objects: 100% (30/30), done.
remote: Total 50 (delta 20), reused 0 (delta 0), pack-reused 0
Unpacking objects: 100% (50/50), done.
From https://github.com/152zbw/my-website
   6bdc204..5c65b8f  main       -> origin/main
Updating 6bdc204..5c65b8f
Fast-forward
 html/single-service.html | 150 ++++++++++++++++++++++++++++++++--------
 html/index.html          | 200 ++++++++++++++++++++++++++++++++++++++++
 ... (更多文件)
 50 files changed, 8318 insertions(+), 1569 deletions(-)
```

**说明：**
- `Fast-forward` 表示更新成功，没有冲突
- 会显示更新的文件列表和统计信息

**如果出现冲突：**

如果显示类似这样的信息：
```
Auto-merging html/index.html
CONFLICT (content): Merge conflict in html/index.html
```

**解决方法：**
1. 查看冲突文件：`git status`
2. 手动解决冲突（编辑文件，删除冲突标记）
3. 或者回滚：`git merge --abort`（放弃本次更新）

### 步骤 5：检查是否有新的依赖

在服务器终端中输入：

```bash
npm install
```

**说明：**
- `npm install` 是安装项目依赖的命令
- 如果 `package.json` 文件有变化，需要运行这个命令
- 如果只是修改了代码文件（HTML、JS等），可以跳过这一步

**预期输出：**
```
npm WARN deprecated xxxx@1.0.0: This package is deprecated
added 5 packages, removed 2 packages, and updated 10 packages in 15s
```

**说明：**
- `added 5 packages` 表示新增了 5 个依赖包
- `removed 2 packages` 表示移除了 2 个依赖包
- `updated 10 packages` 表示更新了 10 个依赖包

**如果没有新依赖：**
```
up to date, audited 150 packages in 2s
```

**说明：** 表示依赖已经是最新的，不需要更新

### 步骤 6：重启应用

在服务器终端中输入：

```bash
pm2 restart my-website
```

**说明：**
- `pm2` 是 Node.js 进程管理工具
- `restart` 是重启命令
- `my-website` 是应用的名称（如果不同，请替换为实际名称）

**预期输出：**
```
[PM2] Applying action restartProcessId on app [my-website] (ids: 0)
[PM2] [my-website] Restarting
[PM2] [my-website] Successfully restarted
```

**说明：**
- `Successfully restarted` 表示重启成功

**如果应用名称不同：**

1. 查看所有应用：`pm2 list`
2. 找到你的应用名称
3. 使用正确的名称重启：`pm2 restart 应用名称`

### 步骤 7：检查运行状态

在服务器终端中输入：

```bash
pm2 status
```

**预期输出：**
```
┌─────┬──────────────┬─────────┬─────────┬──────────┬─────────┐
│ id  │ name         │ status  │ restart │ uptime   │ memory  │
├─────┼──────────────┼─────────┼─────────┼──────────┼─────────┤
│ 0   │ my-website   │ online  │ 0        │ 5s       │ 45.2mb  │
└─────┴──────────────┴─────────┴─────────┴──────────┴─────────┘
```

**说明：**
- `status` 应该是 `online`（在线）
- 如果显示 `errored` 或 `stopped`，说明有问题，需要查看日志

**如果状态不是 `online`：**

查看错误日志：
```bash
pm2 logs my-website --err --lines 50
```

### 步骤 8：查看应用日志

在服务器终端中输入：

```bash
pm2 logs my-website --lines 50
```

**说明：**
- `pm2 logs` 是查看日志的命令
- `--lines 50` 表示只显示最近 50 行日志

**预期输出：**
```
[2024-01-15 14:30:22] [my-website] Server running on port 3000
[2024-01-15 14:30:22] [my-website] Database connected successfully
[2024-01-15 14:30:22] [my-website] Routes loaded
```

**说明：**
- 应该看到服务器启动成功的消息
- 不应该有红色的错误信息

**如果有错误：**

1. 查看完整错误信息
2. 检查代码是否有语法错误
3. 检查数据库连接是否正常
4. 如果问题严重，考虑回滚

**退出日志查看：** 按 `Ctrl + C`

在浏览器中访问你的网站：
- 如果使用 IP：`http://你的服务器IP:3000`
- 如果使用域名：`http://你的域名`

### 2. 测试新功能

根据你修改的内容，测试相关功能：

```bash
pm2 logs my-website --lines 20
```

## 常见问题处理

### ❌ 问题 0：git push 后出现空白/卡住

**现象：**
- 执行 `git push` 后终端没有任何输出，光标闪烁但没有任何提示
- 命令似乎卡住了

**原因：**
- Git 正在等待输入用户名和密码，但提示信息没有显示
- 或者钥匙串中的凭证已过期

**解决方法：**

**方法 A：直接输入用户名和密码（推荐）**

1. **在空白处直接输入你的 GitHub 用户名**
   - 虽然看不到提示，但可以直接输入
   - 输入完成后按回车

2. **然后输入密码或 Personal Access Token**
   - 输入时不会显示任何字符（这是正常的）
   - 输入完成后按回车

3. **如果成功，会看到类似输出：**
   ```
   Username for 'https://github.com': 你的用户名
   Password for 'https://github.com': 
   Enumerating objects: 50, done.
   ...
   To https://github.com/152zbw/my-website.git
      6bdc204..5c65b8f  main -> main
   ```

**方法 B：使用 Personal Access Token（推荐，更安全）**

如果直接输入密码不行，需要使用 Personal Access Token：

1. **生成 Personal Access Token：**
   - 访问：https://github.com/settings/tokens
   - 点击 "Generate new token" → "Generate new token (classic)"
   - 设置名称（如：my-website-push）
   - 选择过期时间（建议选择较长时间）
   - 勾选 `repo` 权限
   - 点击 "Generate token"
   - **重要：立即复制 Token（只显示一次）**

2. **使用 Token 推送：**
   ```bash
   git push
   ```
   - Username: 输入你的 GitHub 用户名
   - Password: **粘贴刚才复制的 Token**（不是账户密码）

3. **保存到钥匙串（可选）：**
   - 第一次输入后，macOS 会询问是否保存到钥匙串
   - 选择 "始终允许" 或 "允许"，以后就不需要再输入了

**方法 C：清除旧的凭证，重新输入**

如果钥匙串中的凭证过期了：

1. **打开钥匙串访问（Keychain Access）：**
   - 按 `Command + 空格`，输入 "钥匙串访问"
   - 或者：应用程序 → 实用工具 → 钥匙串访问

2. **搜索 "github.com"：**
   - 在搜索框输入 "github.com"
   - 找到相关的凭证条目

3. **删除旧的凭证：**
   - 右键点击 → 删除
   - 或者选中后按 `Delete` 键

4. **重新推送：**
   ```bash
   git push
   ```
   - 这次会重新提示输入用户名和密码/Token

**方法 D：使用 SSH 方式（一劳永逸）**

如果经常遇到这个问题，建议改用 SSH：

1. **检查是否已有 SSH 密钥：**
   ```bash
   ls -la ~/.ssh
   ```
   - 如果看到 `id_rsa` 和 `id_rsa.pub`，说明已有密钥

2. **如果没有，生成 SSH 密钥：**
   ```bash
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
   ```
   - 按回车使用默认路径
   - 可以设置密码，也可以直接回车跳过

3. **查看公钥：**
   ```bash
   cat ~/.ssh/id_rsa.pub
   ```
   - 复制输出的全部内容

4. **添加到 GitHub：**
   - 访问：https://github.com/settings/keys
   - 点击 "New SSH key"
   - Title: 输入名称（如：MacBook）
   - Key: 粘贴刚才复制的公钥
   - 点击 "Add SSH key"

5. **修改 Git 远程地址为 SSH：**
   ```bash
   cd "/Users/zade/Desktop/JW 2"
   git remote set-url origin git@github.com:152zbw/my-website.git
   ```

6. **测试连接：**
   ```bash
   ssh -T git@github.com
   ```
   - 如果看到 "Hi 152zbw! You've successfully authenticated..." 说明成功

7. **现在推送就不需要输入密码了：**
   ```bash
   git push
   ```

**如何判断命令是否卡住：**

- **正常情况：** 光标在闪烁，等待输入
- **真的卡住：** 按 `Ctrl + C` 可以中断，说明命令在运行
- **如果按 `Ctrl + C` 没反应：** 可能是终端问题，尝试关闭重新打开

**快速测试：**

如果想知道是否在等待输入，可以：
1. 直接输入你的 GitHub 用户名
2. 按回车
3. 输入密码/Token
4. 按回车
5. 如果成功，会看到推送的输出

### ❌ 问题 0.5：git push 失败 - 连接被拒绝

**错误信息：**
```
fatal: unable to access 'https://github.com/xxx.git/': 
Failed to connect to github.com port 443 after xxx ms: Connection refused
```

**原因：**
- GitHub 的 HTTPS 端口（443）可能被限制或无法访问
- 网络环境问题（防火墙、代理等）

**解决方法：**

**方法 A：使用 SSH 方式（最推荐，最稳定）**

SSH 使用端口 22，通常比 HTTPS 更稳定：

1. **检查是否已有 SSH 密钥：**
   ```bash
   ls -la ~/.ssh
   ```
   - 如果看到 `id_rsa` 和 `id_rsa.pub`，说明已有密钥，跳到第 3 步

2. **如果没有，生成 SSH 密钥：**
   ```bash
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
   ```
   - 按回车使用默认路径（`~/.ssh/id_rsa`）
   - 可以设置密码，也可以直接回车跳过
   - 再次回车确认密码

3. **查看公钥：**
   ```bash
   cat ~/.ssh/id_rsa.pub
   ```
   - 复制输出的全部内容（从 `ssh-rsa` 开始到邮箱结束）

4. **添加到 GitHub：**
   - 访问：https://github.com/settings/keys
   - 点击 "New SSH key"
   - Title: 输入名称（如：MacBook）
   - Key: 粘贴刚才复制的公钥内容
   - 点击 "Add SSH key"

5. **测试 SSH 连接：**
   ```bash
   ssh -T git@github.com
   ```
   - 第一次会提示确认，输入 `yes` 并按回车
   - 如果看到 "Hi 152zbw! You've successfully authenticated..." 说明成功

6. **修改 Git 远程地址为 SSH：**
   ```bash
   cd "/Users/zade/Desktop/JW 2"
   git remote set-url origin git@github.com:152zbw/my-website.git
   ```

7. **验证远程地址：**
   ```bash
   git remote -v
   ```
   - 应该显示 `git@github.com:152zbw/my-website.git`（SSH 格式）

8. **现在推送：**
   ```bash
   git push
   ```
   - 使用 SSH 方式通常不需要输入密码（如果密钥有密码，会提示输入密钥密码）

**方法 B：配置 Git 使用代理（如果你有代理）**

如果你有可用的代理（VPN、Clash 等）：

1. **查看代理端口：**
   - 通常代理软件会显示端口，如：`127.0.0.1:7890` 或 `127.0.0.1:1087`

2. **配置 Git 使用代理：**
   ```bash
   # HTTP 代理（适用于 HTTP/HTTPS）
   git config --global http.proxy http://127.0.0.1:7890
   git config --global https.proxy http://127.0.0.1:7890
   
   # 或者 SOCKS5 代理
   git config --global http.proxy socks5://127.0.0.1:7890
   git config --global https.proxy socks5://127.0.0.1:7890
   ```
   - 将 `7890` 替换为你的实际代理端口

3. **测试连接：**
   ```bash
   git push
   ```

4. **如果不再需要代理，可以取消：**
   ```bash
   git config --global --unset http.proxy
   git config --global --unset https.proxy
   ```

**方法 C：使用 GitHub 镜像（临时方案）**

如果以上方法都不行，可以尝试使用镜像：

1. **修改 hosts 文件（需要管理员权限）：**
   ```bash
   sudo nano /etc/hosts
   ```
   - 添加以下行：
     ```
     140.82.112.3 github.com
     140.82.112.4 github.com
     ```
   - 按 `Ctrl + O` 保存，`Ctrl + X` 退出

2. **刷新 DNS：**
   ```bash
   sudo dscacheutil -flushcache
   sudo killall -HUP mDNSResponder
   ```

3. **重新尝试推送：**
   ```bash
   git push
   ```

**方法 D：使用 Gitee 作为中转（如果 GitHub 无法访问）**

如果 GitHub 完全无法访问，可以使用 Gitee 作为中转：

1. **在 Gitee 创建仓库并同步 GitHub：**
   - 访问：https://gitee.com
   - 创建新仓库
   - 在仓库设置中配置 GitHub 同步

2. **添加 Gitee 为远程仓库：**
   ```bash
   git remote add gitee https://gitee.com/你的用户名/my-website.git
   ```

3. **推送到 Gitee：**
   ```bash
   git push gitee main
   ```

**推荐顺序：**

1. **首选：使用 SSH 方式**（方法 A）- 最稳定，不需要代理
2. **如果有代理：配置代理**（方法 B）
3. **临时方案：修改 hosts**（方法 C）
4. **最后选择：使用 Gitee 中转**（方法 D）

### ❌ 问题 1：git pull 失败 - 需要认证

**错误信息：**
```
Permission denied (publickey).
fatal: Could not read from remote repository.
```

**解决方法：**

**方法 A：配置 SSH 密钥（推荐）**

1. 在本地生成 SSH 密钥：
   ```bash
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
   ```
   - 按回车使用默认路径
   - 可以设置密码，也可以直接回车跳过

2. 查看公钥：
   ```bash
   cat ~/.ssh/id_rsa.pub
   ```
   - 复制输出的内容

3. 在 GitHub/Gitee 添加 SSH 密钥：
   - GitHub: Settings → SSH and GPG keys → New SSH key
   - Gitee: 设置 → SSH公钥 → 添加公钥
   - 粘贴刚才复制的公钥内容

4. 在服务器上配置 Git 使用 SSH：
   ```bash
   cd /root/my-website
   git remote -v  # 查看当前远程地址
   git remote set-url origin git@github.com:152zbw/my-website.git  # 改为 SSH 地址
   ```

**方法 B：使用 Personal Access Token**

1. 在 GitHub/Gitee 生成 Token
2. 在服务器上使用 Token 作为密码：
   ```bash
   git pull
   # 用户名：你的 GitHub 用户名
   # 密码：输入 Token（不是账户密码）
   ```

### ❌ 问题 2：git pull 失败 - 有未提交的修改

**错误信息：**
```
error: Your local changes to the following files would be overwritten by merge:
    html/index.html
Please commit your changes or stash them before you merge.
```

**解决方法：**

**方法 A：保存本地修改（推荐）**
```bash
# 保存当前修改
git stash

# 拉取最新代码
git pull

# 恢复之前的修改（如果需要）
git stash pop
```

**方法 B：放弃本地修改**
```bash
# 放弃所有本地修改
git reset --hard

# 拉取最新代码
git pull
```

**⚠️ 注意：** 方法 B 会永久删除服务器上的本地修改！

### ❌ 问题 5：应用启动失败

**错误信息：**
```
[PM2] [my-website] Status: errored
```

**解决方法：**

1. **查看详细错误日志：**
   ```bash
   pm2 logs my-website --err --lines 100
   ```

2. **常见错误和解决方法：**

   **错误：端口被占用**
   ```
   Error: listen EADDRINUSE: address already in use :::3000
   ```
   **解决：**
   ```bash
   # 查找占用端口的进程
   lsof -i :3000
   # 或者
   netstat -tulpn | grep 3000
   
   # 杀死进程
   kill -9 进程ID
   
   # 重新启动
   pm2 restart my-website
   ```

   **错误：数据库连接失败**
   ```
   Error: SQLITE_CANTOPEN: unable to open database file
   ```
   **解决：**
   ```bash
   # 检查数据库文件是否存在
   ls -la data/database.sqlite
   
   # 检查文件权限
   chmod 644 data/database.sqlite
   
   # 检查目录权限
   chmod 755 data
   ```

   **错误：模块找不到**
   ```
   Error: Cannot find module 'express'
   ```
   **解决：**
   ```bash
   # 重新安装依赖
   npm install
   ```

### ❌ 问题 6：更新后网站无法访问

**解决方法：**

1. **检查 PM2 状态：**
   ```bash
   pm2 status
   ```
   - 如果状态不是 `online`，查看日志找出问题

2. **检查端口是否监听：**
   ```bash
   netstat -tulpn | grep 3000
   ```
   - 应该看到类似 `0.0.0.0:3000` 的输出

3. **检查防火墙：**
   ```bash
   # 查看防火墙规则
   firewall-cmd --list-all
   # 或者
   iptables -L -n
   ```

4. **检查 Nginx（如果使用了反向代理）：**
   ```bash
   # 检查 Nginx 状态
   systemctl status nginx
   
   # 检查 Nginx 配置
   nginx -t
   
   # 重新加载配置
   systemctl reload nginx
   ```

### ❌ 问题 7：更新后功能不正常

**解决方法：**

1. **清除浏览器缓存：**
   - 按 `Ctrl + Shift + Delete`（Windows）或 `Cmd + Shift + Delete`（Mac）
   - 选择"缓存的图片和文件"
   - 点击"清除数据"
   - 或者使用无痕模式测试

2. **检查浏览器控制台：**
   - 按 `F12` 打开开发者工具
   - 查看 `Console` 标签的错误信息
   - 查看 `Network` 标签的网络请求

3. **检查服务器日志：**
   ```bash
   pm2 logs my-website --lines 100
   ```

4. **回滚到之前版本（如果问题严重）：**
   ```bash
   cd /root/my-website
   git log  # 查看提交历史
   git checkout 之前的提交ID  # 回滚
   pm2 restart my-website
   ```

## 注意事项

1. **更新前一定要备份数据库！**
   ```bash
   cp data/database.sqlite data/database.sqlite.backup.$(date +%Y%m%d_%H%M%S)
   ```

2. **不要在服务器上直接修改代码**
   - 应该在本地修改、测试
   - 然后推送到远程仓库
   - 最后在服务器上拉取

3. **更新后要测试所有功能**
   - 不要只测试修改的部分
   - 确保其他功能没有受到影响

4. **如果更新失败，立即回滚**
   - 不要尝试修复，先回滚保证网站可用
   - 然后在本地修复问题，重新更新

5. **定期备份数据库**
   - 建议每天备份一次
   - 保留最近 7 天的备份
