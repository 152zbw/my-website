## 第一步：购买阿里云服务器

### 1.1 注册阿里云账号

1. **访问阿里云官网**
   - 打开浏览器，访问：https://www.aliyun.com
   - 点击右上角 **免费注册**

2. **注册账号**
   - 填写手机号、密码等信息
   - 完成实名认证（需要身份证）
   - ⚠️ **重要**：必须完成实名认证才能购买服务器

### 1.2 购买云服务器ECS

#### 方式一：购买新服务器（推荐）

1. **进入云服务器页面**
   - 登录阿里云控制台
   - 在顶部搜索框输入：**云服务器ECS**
   - 点击进入云服务器ECS控制台

2. **立即购买**
   - 点击右上角 **立即购买** 或 **创建实例** 按钮

3. **选择付费方式**
   - **包年包月**：按月或按年付费，更便宜（推荐新手）
   - **按量付费**：按小时付费，适合测试

4. **选择地域和可用区**
   - **地域**：选择离你最近的（如：华东1-杭州、华北2-北京）
   - **可用区**：选择默认即可
   - 💡 **提示**：地域选择会影响访问速度，选择离你近的

5. **选择实例规格**
   
   **配置推荐（够用即可）：**
   
   | 配置 | 说明 | 价格（约） |
   |------|------|-----------|
   | **1核2GB** | 适合小型网站，推荐 | ¥50-80/月 |
   | **2核4GB** | 性能更好，适合中型网站 | ¥100-150/月 |
   | **1核1GB** | 最便宜，但可能不够用 | ¥30-50/月 |
   
   **选择建议：**
   - 如果你是测试：选择 **1核1GB**
   - 如果正式使用：选择 **1核2GB**（推荐）
   - 如果访问量大：选择 **2核4GB**

6. **选择镜像（操作系统）**
   
   **推荐选择：**
   - **Ubuntu 22.04 64位**（推荐，容易操作）
   - 或 **CentOS 7.9 64位**（稳定，但命令稍有不同）
   
   💡 **提示**：选择 Ubuntu 22.04，本教程以此为例

7. **选择存储**
   - **系统盘**：选择 **40GB**（默认即可，够用）
   - **云盘类型**：选择 **高效云盘**（够用，便宜）

8. **配置网络**
   - **专有网络VPC**：选择默认即可
   - **公网IP**：选择 **分配公网IPv4地址**（必须！）
   - **带宽**：选择 **1Mbps**（够用，约¥23/月）
     - 如果访问量大，可以选择 2Mbps 或 3Mbps

9. **设置登录凭证**
   
   ⚠️ **非常重要！记住这些信息！**
   
   **方式一：设置密码（推荐）**
   - 选择 **自定义密码**
   - 设置一个强密码（8-30个字符，包含大小写字母、数字、特殊字符）
   - 📝 **记录下这个密码！**
   
   **方式二：使用密钥对**
   - 如果熟悉SSH密钥，可以选择
   - 新手建议用密码

10. **确认订单**
    - 检查配置是否正确
    - 选择购买时长（1个月、3个月、1年等）
    - 点击 **确认订单** 并支付

#### 方式二：使用免费试用（限时活动）

阿里云有时会提供免费试用（1个月免费），可以关注：
- 阿里云官网首页
- 学生优惠（需要学生认证）

### 1.3 获取服务器信息

购买成功后，你需要记录以下信息：

1. **找到你的服务器**
   - 进入 **云服务器ECS控制台**
   - 点击 **实例** 菜单
   - 你会看到你的服务器列表

2. **记录重要信息**
   
   📝 **请记录以下信息：**
   
   ```
   公网IP地址：47.xxx.xxx.xxx  （这是你服务器的地址）
   内网IP地址：172.xxx.xxx.xxx  （不需要）
   用户名：root                 （通常是root）
   密码：你刚才设置的密码       （非常重要！）
   地域：华东1-杭州             （例如）
   ```

3. **查看服务器状态**
   - 确保状态是 **运行中**（绿色图标）
   - 如果状态是 **已停止**，点击 **启动**

## 第二步：连接服务器

连接服务器有几种方式，推荐使用 **PuTTY**（Windows）或 **终端**（Mac）。

### 2.1 Windows 用户：使用 PuTTY

#### 下载并安装 PuTTY

1. **下载 PuTTY**
   - 访问：https://www.putty.org
   - 点击 **Download PuTTY**
   - 下载 `putty.exe`（约 3MB）

2. **运行 PuTTY**
   - 双击 `putty.exe` 打开
   - 不需要安装，直接使用

#### 连接服务器

1. **填写连接信息**
   ```
   Host Name (or IP address): 47.xxx.xxx.xxx  （输入你的公网IP）
   Port: 22                                    （默认22，不要改）
   Connection type: SSH                        （选择SSH）
   ```

2. **保存配置（可选）**
   - 在 **Saved Sessions** 中输入名称（如：我的服务器）
   - 点击 **Save**
   - 下次可以直接双击名称连接

3. **打开连接**
   - 点击右下角 **Open** 按钮

4. **安全警告**
   - 第一次连接会弹出安全警告
   - 点击 **是** 或 **Accept**

5. **登录**
   - **login as:** 输入 `root`，按回车
   - **password:** 输入你的密码，按回车
     - ⚠️ **注意**：输入密码时屏幕上不会显示，这是正常的，直接输入后按回车

6. **连接成功标志**
   
   看到类似这样的提示，说明连接成功：
   ```
   Welcome to Ubuntu 22.04.3 LTS
   root@iZxxx:~#
   ```
   
   💡 **提示**：`root@iZxxx:~#` 这个提示符表示你可以输入命令了

### 2.2 Mac 用户：使用终端

1. **打开终端**
   - 按 `Command + 空格` 打开 Spotlight
   - 输入 `终端` 或 `Terminal`
   - 点击打开

2. **连接服务器**
   ```bash
   ssh root@你的公网IP
   ```
   
   例如：
   ```bash
   ssh root@47.xxx.xxx.xxx
   ```

3. **安全提示**
   - 第一次连接会提示是否信任，输入 `yes` 并按回车

4. **输入密码**
   - **password:** 输入你的服务器密码
   - ⚠️ **注意**：输入时不会显示，直接输入后按回车

5. **连接成功**
   - 看到 `root@iZxxx:~#` 提示符，说明连接成功

### 2.3 如果连接失败

**常见问题：**

1. **连接超时**
   - 检查公网IP是否正确
   - 检查服务器是否已启动（状态是否为"运行中"）
   - 检查防火墙规则（见第七步）

2. **密码错误**
   - 确认密码是否正确（注意大小写）
   - 如果不记得密码，可以在阿里云控制台重置密码：
     - 实例 → 更多 → 密码/密钥 → 重置实例密码

3. **权限被拒绝**
   - 确认用户名是否正确（通常是 `root`）
   - 确认端口是否正确（SSH默认是22端口）

## 第三步：在服务器上安装环境

现在你已经连接到服务器了，接下来需要安装 Node.js 等运行环境。

### 3.1 更新系统软件（必须）

在 PuTTY 或终端中输入以下命令：

```bash
apt update && apt upgrade -y
```

**说明：**
- `apt update`：更新软件包列表
- `apt upgrade -y`：升级所有软件包（`-y` 表示自动确认）
- 这个过程可能需要 5-10 分钟，请耐心等待
- 如果提示输入 `Y`，输入 `Y` 并按回车

### 3.2 安装 Node.js

我们使用 NodeSource 来安装最新版本的 Node.js：

```bash
# 下载并安装 NodeSource 仓库
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -

# 安装 Node.js
apt-get install -y nodejs
```

**执行过程：**
1. 第一行命令会下载安装脚本并运行（可能需要 1-2 分钟）
2. 第二行命令会安装 Node.js（可能需要 2-3 分钟）
3. 如果看到错误信息，不要慌，继续往下看

### 3.3 验证安装

安装完成后，验证是否安装成功：

```bash
node -v
npm -v
```

**预期输出：**
```
v18.x.x    （Node.js 版本，应该是 v18 或更高）
9.x.x      （npm 版本，应该是 9 或更高）
```

**如果显示版本号，说明安装成功！**

**如果显示 `command not found`：**

说明安装失败，尝试以下方法：

#### 方法二：使用 NVM 安装（推荐备用方案）

```bash
# 安装 NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重新加载配置
source ~/.bashrc

# 安装 Node.js 18
nvm install 18

# 使用 Node.js 18
nvm use 18

# 验证
node -v
npm -v
```

### 3.4 安装 PM2（进程管理工具）

PM2 可以让你的网站一直运行，即使你关闭终端也不会停止：

```bash
npm install -g pm2
```

**验证安装：**
```bash
pm2 -v
```

应该显示版本号，如 `5.x.x`

## 第四步：上传项目文件

现在需要把你电脑上的项目文件上传到服务器。有两种方法：

### 方法一：使用 Git（推荐，最简单）

#### 4.1 在你的电脑上准备代码

**Windows 用户：**

1. **下载 Git**
   - 访问：https://git-scm.com/download/win
   - 下载并安装 Git（默认选项即可）

2. **打开 Git Bash**
   - 在项目文件夹（`JW 2`）中右键
   - 选择 **Git Bash Here**

3. **初始化 Git 仓库**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```

**Mac 用户：**

1. **打开终端**
   - 进入项目文件夹
   ```bash
   cd ~/Desktop/JW\ 2
   ```

2. **初始化 Git 仓库**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```

#### 4.2 上传到 GitHub

1. **注册 GitHub 账号**（如果没有）
   - 访问：https://github.com
   - 点击 **Sign up** 注册

2. **创建仓库**
   - 登录 GitHub
   - 点击右上角 **+** → **New repository**
   - Repository name：输入名称（如：`my-website`）
   - 选择 **Public** 或 **Private**（Private 需要付费，Public 免费）
   - ⚠️ **不要勾选** "Initialize this repository with a README"
   - 点击 **Create repository**

3. **上传代码**
   
   在你的电脑上执行（替换 `你的用户名` 和 `仓库名`）：
   ```bash
   git branch -M main
   git remote add origin https://github.com/你的用户名/仓库名.git
   git push -u origin main
   ```
   
   如果提示输入用户名和密码：
   - **用户名**：你的 GitHub 用户名
   - **密码**：使用 **Personal Access Token**（不是登录密码）
     - 生成 Token：GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token
     - 勾选 `repo` 权限
     - 复制生成的 Token，作为密码使用

#### 4.3 在服务器上克隆代码

回到 PuTTY 或终端（连接到服务器的窗口），执行：

```bash
# 进入用户目录
cd /root

# 安装 Git（如果还没安装）
apt install git -y

# 克隆项目（替换为你的GitHub地址）
git clone https://github.com/你的用户名/仓库名.git

# 进入项目目录
cd 仓库名
```

**例如：**
```bash
cd /root
apt install git -y
git clone https://github.com/zhangsan/my-website.git
cd my-website
```

### 方法二：使用 FTP 工具（适合文件较大时）

#### 4.1 下载 FileZilla

1. **下载 FileZilla**
   - Windows：https://filezilla-project.org/download.php?type=client
   - Mac：同样访问上面的链接，选择 Mac 版本

2. **安装 FileZilla**
   - 下载后安装（默认选项即可）

#### 4.2 连接服务器

1. **打开 FileZilla**
2. **填写连接信息**
   ```
   主机(H): sftp://47.xxx.xxx.xxx    （你的公网IP，前面加 sftp://）
   用户名: root
   密码: 你的服务器密码
   端口: 22
   ```
3. **点击快速连接**

#### 4.3 上传文件

1. **左侧窗口**：你电脑上的项目文件夹（`JW 2`）
2. **右侧窗口**：服务器的 `/root` 目录
3. **拖拽**：把整个项目文件夹拖到右侧
4. **等待上传完成**（可能需要几分钟）

## 第五步：配置和启动项目

### 5.1 进入项目目录

在 PuTTY 或终端中执行：

```bash
cd /root/my-website
# 或者如果你用FTP上传的
cd /root/JW\ 2
```

### 5.2 安装项目依赖

```bash
npm install
```

**说明：**
- 这会安装项目所需的所有依赖包
- 可能需要 2-5 分钟
- 如果看到警告（WARN），可以忽略
- 如果看到错误（ERROR），需要检查网络连接

### 5.3 创建环境变量文件

```bash
nano .env
```

**说明：**
- `nano` 是一个文本编辑器
- 会打开一个编辑窗口

**在编辑器中输入以下内容**（按你的需求修改）：

```env
PORT=3000
NODE_ENV=production
JWT_SECRET=你的随机密钥至少32个字符建议用强密码
JWT_EXPIRES_IN=24h
DB_DIALECT=sqlite
```

**生成随机密钥的方法**：

在服务器上执行：
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

复制输出的字符串，作为 `JWT_SECRET` 的值。

**保存文件：**
1. 按 `Ctrl + X`（退出编辑器）
2. 按 `Y`（确认保存）
3. 按 `Enter`（确认文件名）

### 5.4 初始化数据库

```bash
DB_DIALECT=sqlite node init-db.js
```

**说明：**
- 这会创建数据库表并插入初始数据
- 如果看到 "数据库初始化完成！" 说明成功

**预期输出：**
```
开始初始化数据库...
数据库模型同步成功
默认管理员用户创建成功
用户名: admin
密码: admin123
示例导航项创建成功
...
数据库初始化完成！
```

### 5.5 测试启动

先测试一下能否正常启动：

```bash
node app.js
```

**预期输出：**
```
使用 SQLite 数据库: /root/my-website/data/database.sqlite
数据库连接成功！
服务器正在运行，端口: 3000
访问地址: http://localhost:3000
API地址: http://localhost:3000/api
```

**如果看到这些信息，说明启动成功！**

⚠️ **注意**：现在服务器在运行，但你不能关闭这个窗口。按 `Ctrl + C` 停止服务器。

## 第六步：使用PM2保持运行

现在需要让服务器在后台一直运行，即使你关闭终端也不会停止。

### 6.1 使用 PM2 启动

```bash
cd /root/my-website  # 确保在项目目录

# 使用 PM2 启动（设置环境变量）
DB_DIALECT=sqlite pm2 start app.js --name my-website
```

**说明：**
- `my-website` 是应用的名称，可以改成你喜欢的名字
- 启动后，服务器会在后台运行

### 6.2 查看状态

```bash
pm2 status
```

**预期输出：**
```
┌─────┬──────────────┬─────────┬─────────┬──────────┬─────────┐
│ id  │ name         │ mode    │ ↺       │ status   │ cpu     │
├─────┼──────────────┼─────────┼─────────┼──────────┼─────────┤
│ 0   │ my-website   │ cluster │ 0       │ online   │ 0%      │
└─────┴──────────────┴─────────┴─────────┴──────────┴─────────┘
```

**如果 status 是 `online`，说明运行正常！**

### 6.3 设置开机自启

让服务器重启后自动启动你的网站：

```bash
# 生成启动脚本
pm2 startup

# 保存当前进程列表
pm2 save
```

**说明：**
- `pm2 startup` 会输出一行命令，**复制并执行这行命令**（通常是 `sudo env PATH=... pm2 startup ...`）
- `pm2 save` 保存当前运行的进程

### 6.4 常用 PM2 命令

```bash
pm2 status              # 查看状态
pm2 logs my-website     # 查看日志
pm2 restart my-website  # 重启应用
pm2 stop my-website     # 停止应用
pm2 delete my-website   # 删除应用
```

## 第七步：配置防火墙

现在需要开放端口，让外网可以访问你的网站。

### 7.1 在阿里云控制台配置安全组

1. **进入安全组设置**
   - 登录阿里云控制台
   - 进入 **云服务器ECS** → **实例**
   - 点击你的服务器实例
   - 点击 **安全组** 标签
   - 点击安全组ID（如：`sg-xxx`）

2. **添加安全组规则**
   - 点击 **入方向** 标签
   - 点击 **添加安全组规则**

3. **配置规则**
   ```
   端口范围: 3000/3000
   授权对象: 0.0.0.0/0
   描述: 允许访问网站端口
   ```
   - 点击 **保存**

### 7.2 在服务器上配置防火墙

在 PuTTY 或终端中执行：

```bash
# 检查防火墙状态
ufw status

# 如果防火墙未启用，启用它
ufw enable

# 允许 3000 端口
ufw allow 3000/tcp

# 确认规则已添加
ufw status numbered
```

**预期输出：**
```
Status: active

     To                         Action      From
     --                         ------      ----
[ 1] 3000/tcp                   ALLOW IN    Anywhere
```

## 第八步：配置域名（可选）

如果你有自己的域名，可以让用户通过域名访问你的网站。

### 8.1 购买域名

1. **在阿里云购买域名**
   - 登录阿里云控制台
   - 搜索 **域名** → **域名注册**
   - 输入你想要的域名（如：`mywebsite.com`）
   - 查看价格并购买

2. **实名认证**（必须）
   - 购买后需要实名认证
   - 按照提示完成认证

### 8.2 解析域名

1. **进入域名解析**
   - 阿里云控制台 → **域名** → **解析**
   - 找到你购买的域名
   - 点击 **解析设置**

2. **添加A记录**
   - 点击 **添加记录**
   - 配置如下：
     ```
     记录类型: A
     主机记录: @  （表示主域名，如 mywebsite.com）
     记录值: 47.xxx.xxx.xxx  （你的服务器公网IP）
     TTL: 600
     ```
   - 点击 **确认**

3. **添加www记录**（可选）
   - 再次点击 **添加记录**
   - 配置：
     ```
     记录类型: A
     主机记录: www  （表示 www.mywebsite.com）
     记录值: 47.xxx.xxx.xxx
     TTL: 600
     ```
   - 点击 **确认**

4. **等待生效**
   - DNS解析需要 10 分钟到 24 小时
   - 等待后，访问 `http://你的域名:3000` 应该能看到网站

## 第九步：使用Nginx反向代理（可选）

使用 Nginx 可以让你的网站：
- 使用 80 端口（不用加 `:3000`）
- 使用 HTTPS（更安全）
- 提高性能

### 9.1 安装 Nginx

```bash
apt install nginx -y
```

### 9.2 配置 Nginx

```bash
nano /etc/nginx/sites-available/my-website
```

**在编辑器中输入以下内容**（替换 `yourdomain.com` 为你的域名或IP）：

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**保存文件**（`Ctrl + X` → `Y` → `Enter`）

### 9.3 启用配置

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/my-website /etc/nginx/sites-enabled/

# 测试配置是否正确
nginx -t

# 重启 Nginx
systemctl restart nginx

# 设置开机自启
systemctl enable nginx
```

### 9.4 开放 80 端口

1. **在阿里云安全组添加规则**（同第七步）
   - 端口：`80/80`
   - 授权对象：`0.0.0.0/0`

2. **在服务器防火墙开放端口**
   ```bash
   ufw allow 80/tcp
   ```

### 9.5 测试访问

现在可以通过以下方式访问：
- `http://你的域名`（如果配置了域名）
- `http://你的IP`（如果没配置域名）

**不需要加 `:3000` 了！**

## 常见问题解决

### ❌ 问题1：网站打不开

**检查步骤：**

1. **检查服务是否运行**
   ```bash
   pm2 status
   ```
   如果状态不是 `online`，重启：
   ```bash
   pm2 restart my-website
   ```

2. **检查端口是否开放**
   ```bash
   # 在服务器上检查
   netstat -tlnp | grep 3000
   
   # 在阿里云控制台检查安全组规则
   ```

3. **检查防火墙**
   ```bash
   ufw status
   ```

4. **查看日志**
   ```bash
   pm2 logs my-website
   ```

### ❌ 问题2：数据库错误

**错误信息：** `数据库连接失败`

**解决方法：**
```bash
# 检查数据库文件是否存在
ls -la data/database.sqlite

# 如果不存在，重新初始化
cd /root/my-website
DB_DIALECT=sqlite node init-db.js

# 检查文件权限
chmod 644 data/database.sqlite
```

### ❌ 问题3：端口被占用

**错误信息：** `EADDRINUSE: address already in use`

**解决方法：**
```bash
# 查找占用 3000 端口的进程
lsof -i :3000

# 或者
netstat -tlnp | grep 3000

# 杀死进程（替换 PID 为实际进程号）
kill -9 PID

# 或者重启应用
pm2 restart my-website
```

### ❌ 问题4：内存不足

**错误信息：** `JavaScript heap out of memory`

**解决方法：**
```bash
# 增加 Node.js 内存限制
pm2 restart my-website --update-env --node-args="--max-old-space-size=2048"
```

### ❌ 问题5：上传文件失败

**可能原因：** `uploads/` 目录不存在或权限不足

**解决方法：**
```bash
# 进入项目目录
cd /root/my-website

# 创建上传目录
mkdir -p uploads

# 设置权限
chmod 755 uploads
```

### ❌ 问题6：Git 克隆失败

**错误信息：** `Permission denied` 或 `Authentication failed`

**解决方法：**
1. 确认 GitHub 仓库是 **Public**，或
2. 使用 Personal Access Token：
   ```bash
   git clone https://你的token@github.com/用户名/仓库名.git
   ```
