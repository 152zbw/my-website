# 网站部署说明

## 系统要求

- Node.js 14.0 或更高版本
- npm 6.0 或更高版本
- 无需安装数据库（使用SQLite）

## 快速开始

### Windows系统

1. 双击运行 `start.bat` 文件
2. 等待依赖安装和数据库初始化完成
3. 浏览器访问 `http://localhost:3000`

### Mac/Linux系统

1. 打开终端，进入项目目录
2. 运行命令：
   ```bash
   chmod +x start.sh
   ./start.sh
   ```
3. 等待依赖安装和数据库初始化完成
4. 浏览器访问 `http://localhost:3000`

### 手动启动

如果自动启动脚本无法运行，可以手动执行以下步骤：

1. **安装依赖**
   ```bash
   npm install
   ```

2. **创建.env文件**（如果不存在）
   在项目根目录创建 `.env` 文件，内容如下：
   ```
   PORT=3000
   NODE_ENV=development
   JWT_SECRET=your-secret-key-change-this-in-production
   JWT_EXPIRES_IN=24h
   ```

3. **初始化数据库**
   ```bash
   node init-db.js
   ```

4. **启动服务器**
   ```bash
   npm start
   ```

- **前端网站**: http://localhost:3000

## 功能说明

### 后台管理系统

后台管理系统位于 `/html/admin/` 目录，包含以下功能：

1. **仪表盘** - 查看网站统计信息
2. **网站信息** - 管理网站基本信息（标题、描述、联系方式等）
3. **服务项目** - 管理服务项目
4. **成功案例** - 管理成功案例
5. **新闻动态** - 管理新闻/博客文章
6. **客户评价** - 管理客户评价
7. **合作伙伴** - 管理合作伙伴
8. **关于我们** - 管理关于我们内容
9. **团队成员** - 管理团队成员
10. **联系信息** - 查看和管理用户留言

### 前端网站

前端网站会自动从后端API获取数据并显示，包括：

- 首页动态内容（服务、案例、新闻、评价）
- 服务详情页
- 案例详情页
- 新闻/博客列表和详情
- 关于我们页面
- 联系我们页面

## 数据库

项目使用SQLite数据库，数据库文件会自动创建在 `config/database.sqlite`。

**无需安装任何数据库软件**，SQLite是文件型数据库，随项目一起运行。

## 文件上传

上传的文件会保存在 `uploads/` 目录中。

## 常见问题

### 1. 端口被占用

如果3000端口被占用，可以修改 `.env` 文件中的 `PORT` 值。

### 2. 依赖安装失败

如果npm安装依赖失败，可以尝试：
```bash
npm install --registry=https://registry.npmmirror.com
```

### 3. 数据库初始化失败

如果数据库初始化失败，可以删除 `config/database.sqlite` 文件后重新运行 `node init-db.js`。

### 4. 前端无法加载数据

确保：
- 后端服务器正在运行
- 浏览器控制台没有CORS错误
- API地址正确（`http://localhost:3000/api`）

## 生产环境部署

### 使用PM2（推荐）

1. 安装PM2：
   ```bash
   npm install -g pm2
   ```

2. 启动应用：
   ```bash
   pm2 start app.js --name consulting-company
   ```

3. 查看状态：
   ```bash
   pm2 status
   ```

4. 查看日志：
   ```bash
   pm2 logs consulting-company
   ```

### 环境变量配置

在生产环境中，请修改 `.env` 文件：

```
PORT=3000
NODE_ENV=production
JWT_SECRET=请设置一个强密码（至少32个字符）
JWT_EXPIRES_IN=24h
```

### 安全建议

1. **修改JWT密钥**: 在 `.env` 文件中设置一个强密码
2. **修改默认管理员密码**: 首次登录后立即修改
3. **使用HTTPS**: 在生产环境中使用HTTPS协议
4. **定期备份数据库**: 备份 `config/database.sqlite` 文件

## 技术支持

如有问题，请检查：
1. Node.js版本是否符合要求
2. 所有依赖是否已正确安装
3. 端口是否被占用
4. 数据库文件权限是否正确
