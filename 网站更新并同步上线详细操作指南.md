# 网站更新并同步上线详细操作指南（生产环境更新版）

适用场景：
- 你的网站已经部署到服务器并正常上线
- 服务器环境：阿里云 ECS / 宝塔面板 / 自定义 Nginx
- 应用方式：Node.js + Express + PM2 + Nginx 反向代理
- 域名：`ec-int.cn` / `www.ec-int.cn`

> 目标：每次你更新代码后，都能安全、稳定地同步到线上网站，并且尽量避免“更新后打不开”“更新后白屏”“更新后后台异常”等问题。

---

## 0. 先记住一个原则

更新线上网站时，顺序永远建议是：

1. **先备份**
2. **再拉代码**
3. **再安装依赖**
4. **如有数据库变更，先处理数据库**
5. **重启 PM2**
6. **检查 Nginx**
7. **最后做浏览器验收**

不要跳步骤，不要直接覆盖后就立刻刷新网页。

---

## 1. 每次更新前，你要先确认的 3 件事

### 1.1 先确认你能登录服务器

在你本地电脑打开终端，连接服务器：

```bash
ssh root@47.104.69.36
```

如果你的服务器公网 IP 以后变了，就把上面的 IP 换成新的；如果你已经在服务器里了，可以跳过。

### 1.2 先确认线上程序当前是正常的

进入服务器后执行：

```bash
pm2 list
/www/server/nginx/sbin/nginx -t
curl -I http://www.ec-int.cn
```

你希望看到：
- `ec-int` 是 `online`
- Nginx 检查通过
- 网站返回 `200 OK`

### 1.3 先确认你当前目录就是项目目录

一般建议进入：

```bash
cd /var/www/ec-int
pwd
```

如果你的项目不在这个目录，就换成你真实部署目录。

---

## 2. 更新前备份当前版本

这一步很重要。万一新代码有问题，可以快速回滚。

### 2.1 备份代码

```bash
cd /var/www/ec-int
mkdir -p /var/backups/ec-int

tar --exclude='node_modules' -czf /var/backups/ec-int/code-$(date +%F-%H%M%S).tar.gz .
```

### 2.2 备份数据库

如果你使用的是 SQLite，先备份数据库文件：

```bash
cp /var/www/ec-int/data/database.sqlite /var/backups/ec-int/database-$(date +%F-%H%M%S).sqlite
```

如果你的数据库路径不同，就把上面的路径改成你的真实路径。

### 2.3 确认备份成功

```bash
ls -lh /var/backups/ec-int
```

你应该能看到刚刚生成的压缩包和数据库备份。

---

## 3. 拉取最新代码

### 3.1 进入项目目录

```bash
cd /var/www/ec-int
```

### 3.2 拉取远程最新版本

如果你是 Git 部署：

```bash
git fetch --all
git pull
```

如果提示有冲突，先不要硬合并，先看冲突内容再处理。

### 3.3 查看更新内容

建议先看一下本次更新了什么：

```bash
git log --oneline -n 5
git status
```

如果你不确定这次更新是否完整，先不要继续往下。

---

## 4. 安装或更新依赖

### 4.1 安装依赖

```bash
cd /var/www/ec-int
npm install
```

如果项目使用的是 `package-lock.json` 并且你希望依赖更稳定，也可以用：

```bash
npm ci
```

但前提是你的项目适合用 `npm ci`。

### 4.2 检查是否有安装错误

如果安装过程报错，优先查看：
- Node 版本是否正确
- 是否有某个依赖版本不兼容
- 是否缺少系统依赖

常用检查：

```bash
node -v
npm -v
```

---

## 5. 如果这次更新涉及数据库结构，先处理数据库

### 5.1 先判断有没有数据库变更

你需要看这次代码更新是否包含：
- 新表
- 新字段
- 数据迁移脚本
- 初始化脚本变化
- 后台新增/编辑功能依赖的新数据结构

### 5.2 如果项目仍然使用 `init-db`

你可以执行：

```bash
npm run init-db
```

### 5.3 如果项目以后改成真正的迁移脚本

那就应该执行 migration 命令，而不是一直重复 `init-db`。

例如可能是：

```bash
npm run migrate
```

或者：

```bash
npm run db:migrate
```

具体要看你的项目脚本定义。

### 5.4 如果你不确定数据库是否需要更新

先打开 `package.json` 看脚本：

```bash
cat package.json
```

重点看 `scripts` 里有没有：
- `init-db`
- `migrate`
- `db:migrate`
- `reset-db`

---

## 6. 重启 PM2 中的应用

### 6.1 先看当前进程

```bash
pm2 list
```

### 6.2 重启应用

如果你的应用名是 `ec-int`：

```bash
pm2 restart ec-int
```

如果你更新的是整个站点，且你不确定运行中的进程状态，也可以直接重启全部：

```bash
pm2 restart all
```

但生产环境通常更推荐只重启目标应用。

### 6.3 保存进程列表

```bash
pm2 save
```

这一步很重要，不然服务器重启后可能恢复不到最新进程状态。

### 6.4 看重启日志

```bash
pm2 logs ec-int --lines 80
```

如果启动失败，这里通常能直接看到原因。

---

## 7. 检查 Nginx 是否仍然正常

### 7.1 测试 Nginx 配置语法

```bash
/www/server/nginx/sbin/nginx -t
```

### 7.2 重载 Nginx

```bash
/www/server/nginx/sbin/nginx -s reload
```

### 7.3 如果有警告

你可能会看到类似：

```text
conflicting server name ... ignored
```

这通常是你服务器里还有其他旧站点配置重复了 `server_name`。

如果你当前站点访问正常，这个 warning 可以先忽略；但后续最好清理重复配置。

---

## 8. 更新完成后怎么验收

更新完后，不要只看 PM2 在线，还要做浏览器和命令双重确认。

### 8.1 先在服务器上检查

```bash
curl -I https://www.ec-int.cn
curl -I https://www.ec-int.cn/html/index.html
curl -I https://www.ec-int.cn/html/admin/index.html
```

你希望看到：
- `200 OK`
- 或者 HTTP 自动跳转到 HTTPS

### 8.2 再用浏览器打开

依次检查：
- `https://www.ec-int.cn`
- `https://www.ec-int.cn/html/index.html`
- `https://www.ec-int.cn/html/admin/index.html`

### 8.3 登录后台做一次真实操作

建议你至少验证一项：
- 登录后台
- 打开某个管理页面
- 新增或编辑一条内容
- 刷新页面确认数据保存成功

---

## 9. 更新后常见问题排查

### 9.1 更新后页面白屏

先检查：

```bash
pm2 logs ec-int --lines 120
```

再检查浏览器控制台，看看是不是前端 JS 报错。

### 9.2 更新后 502 / 504

通常说明 Node 没起来，或者端口不对。

检查：

```bash
pm2 list
curl -I http://127.0.0.1:3000
```

### 9.3 更新后后台登录失败

检查：
- `JWT_SECRET` 是否改变了
- 认证接口是否返回 401
- Cookie / Token 是否失效
- 域名是否从 HTTP 切到 HTTPS 后导致旧登录态失效

### 9.4 更新后静态资源没刷新

可能是浏览器缓存。

你可以：
- 强刷页面
- 清理浏览器缓存
- 检查静态资源是否有版本号参数

---

## 10. 回滚方案

如果更新后发现严重问题，要立即回滚。

### 10.1 回滚代码

先看最近提交：

```bash
cd /var/www/ec-int
git log --oneline -n 10
```

找到上一个稳定版本的 commit id，然后：

```bash
git reset --hard 选中的commit-id
npm install
pm2 restart ec-int
pm2 save
```

### 10.2 回滚数据库

如果数据库也被破坏了：

```bash
pm2 stop ec-int
cp /var/backups/ec-int/database-YYYY-MM-DD-HHMMSS.sqlite /var/www/ec-int/data/database.sqlite
pm2 start ec-int
pm2 save
```

### 10.3 回滚后重新验证

回滚后再执行：

```bash
curl -I http://www.ec-int.cn
curl -I https://www.ec-int.cn
pm2 list
```

## 11. 推荐的标准更新流程模板

以后你每次更新都可以照这个顺序来：

```bash
cd /var/www/ec-int

# 1. 备份
mkdir -p /var/backups/ec-int
tar --exclude='node_modules' -czf /var/backups/ec-int/code-$(date +%F-%H%M%S).tar.gz .
cp /var/www/ec-int/data/database.sqlite /var/backups/ec-int/database-$(date +%F-%H%M%S).sqlite

# 2. 拉代码
git fetch --all
git pull

# 3. 装依赖
npm install

# 4. 数据库（按需）
# npm run init-db

# 5. 重启服务
pm2 restart ec-int
pm2 save

# 6. 检查配置
/www/server/nginx/sbin/nginx -t
/www/server/nginx/sbin/nginx -s reload

# 7. 验收
curl -I https://www.ec-int.cn
curl -I https://www.ec-int.cn/html/index.html
```

---

## 12. 建议你长期保留的操作习惯

1. **每次更新前先备份**
2. **只改一个版本，不要连续改多次再一起上线**
3. **更新后立刻看 `pm2 logs`**
4. **更新后浏览器真实验收，不要只看命令输出**
5. **数据库有变化时，一定先确认是否需要迁移**
6. **上线前尽量先在测试环境验证**

---

## 13. 一句总结

更新线上网站最稳妥的方式不是“把代码传上去就完事”，而是：

**先备份 → 拉代码 → 装依赖 → 处理数据库 → 重启 PM2 → 检查 Nginx → 浏览器验收 → 有问题就回滚**

只要你每次都按这个流程走，出问题的概率会低很多。
